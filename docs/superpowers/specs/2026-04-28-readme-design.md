## Структура документа

### 1. Шапка

```markdown
# LLM Wiki — плагин Obsidian

Автоматически строит и пополняет wiki-базу знаний из ваших заметок с помощью LLM.

> Поддерживаемые бэкенды: **Ollama / OpenAI-compatible** (без облака) · **Claude Code** (Anthropic)
```

### 2. Что умеет (4 пункта)

- **Ingest** — разбирает заметку, извлекает сущности (люди, технологии, процессы), создаёт и обновляет wiki-страницы
- **Query** — отвечает на вопрос по базе знаний; опционально сохраняет ответ как новую страницу с [[WikiLinks]]
- **Lint** — проверяет качество wiki-домена, находит неполные и устаревшие страницы
- **Init** — инициализирует новый домен с нуля (структура папок, _schema.md, _index.md)

### 3. Quick Start — Native Agent (Ollama)

Шаги:
1. Установить Ollama + `ollama pull llama3.2`
2. Установить плагин (копирование папки или симлинк)
3. Включить в Obsidian → Settings → Community plugins
4. Настроить: Backend = Native Agent, Base URL, API Key, Модель
5. Создать домен (команда Init или вручную через domain-map JSON)
6. Первый Ingest: открыть заметку → Command Palette → «LLM Wiki: Ingest активного файла»

Таблица с конкретными значениями для Ollama (Base URL, API Key, Модель, Temperature=0.2, Max tokens=4096).

### 4. Quick Start — Claude Code

Шаги:
1. Требования: iclaude.sh / claude CLI, скилл llm-wiki
2. Настроить: Backend = Claude Code, путь к iclaude, путь к навыку
3. Выбрать модель (пресеты: opus / sonnet / haiku)
4. Первый Ingest (то же что выше)

Таблица с конкретными значениями для Claude Code (пути-примеры с placeholder /home/user/..., модель sonnet).

### 5. Команды

Таблица:

| Команда | Действие | Результат |
|---|---|---|
| Открыть панель | Показать боковую панель | Живой лог, история операций |
| Ingest активного файла | Извлечь сущности из текущей заметки | Новые/обновлённые wiki-страницы |
| Query | Вопрос по базе знаний | Ответ в панели с [[WikiLinks]] |
| Query + сохранить | Вопрос + сохранить ответ | Новая wiki-страница, открывается автоматически |
| Lint домена | Проверить качество wiki | Отчёт о проблемах в панели |
| Init домена | Инициализировать новый домен | Структура wiki-папок и служебные файлы |
| Отменить операцию | Остановить текущую операцию | SIGTERM → SIGKILL через 3с |

### 6. Справочник настроек

#### 6а. Общие (оба бэкенда)

| Параметр | Описание | По умолчанию |
|---|---|---|
| Backend | claude-code или native-agent | claude-code |
| Лимит истории | Максимум записей в истории панели | 20 |
| Лог агента (JSONL) | Абсолютный путь к файлу лога; пусто — отключено | — |

#### 6б. Native Agent

| Параметр | Описание | По умолчанию |
|---|---|---|
| Base URL | OpenAI-compatible endpoint | http://localhost:11434/v1 |
| API Key | «ollama» для Ollama; sk-... для OpenAI | ollama |
| Модель | Имя модели: llama3.2, mistral, gpt-4o... | llama3.2 |
| Temperature | 0.0–1.0. Низкая (0.1–0.3) = точные факты | 0.2 |
| Max tokens | Макс. токенов в ответе; ≥ 4096 для wiki | 4096 |
| Top-p | Nucleus sampling 0.0–1.0; пусто = отключено | — |
| Request timeout (сек) | Таймаут HTTP; для больших моделей Ollama ≥ 300 | 300 |
| num_ctx | Размер контекста (только Ollama); пусто = дефолт модели | — |
| System prompt | Системный промпт; перезаписывает встроенный | встроенный |

#### 6в. Claude Code

| Параметр | Описание | По умолчанию |
|---|---|---|
| Путь к Claude Code | Полный путь к iclaude.sh / iclaude / claude | — |
| Путь к навыку llm-wiki | Полный путь к папке навыка | — |
| Allowed tools | Список через запятую | Read,Edit,Write,Glob,Grep |
| Модель | Пресет (opus/sonnet/haiku) или произвольный ID | (дефолт claude) |
| Таймауты (ingest/query/lint/init) | Секунды через «/» | 300/300/600/3600 |
| Показывать raw JSON | Отображать сырые JSON-события в панели | выкл |
