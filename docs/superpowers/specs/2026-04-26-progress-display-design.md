# Спек: Улучшенный блок прогресса

**Дата:** 2026-04-26  
**Статус:** Approved  
**Файлы:** `src/view.ts`

## Проблема

Текущий блок прогресса показывает сырые внутренние события Claude Code API:
- `hook_started`, `hook_response`, `init (claude-sonnet-4-6)` — непонятные пользователю строки
- Нет общего ощущения прогресса (сколько шагов, сколько времени)
- Нет возможности скрыть детали, если они не нужны

## Решение: раскрывающийся блок (Вариант B)

Один кликабельный заголовок с раскрывающимся списком шагов внутри.

### Структура UI

```
▼  Ход выполнения   6 шагов · 14.3s        ← кликабельный h4 с ▶/▼
─────────────────────────────────────────
  ⚙ Запуск                          0.1s
  ⚙ Инициализация (claude-sonnet-4-6)  0.3s
  ✔ Read   ai-harness.md            1.2s
  ✔ Grep   "harness"                0.3s
  ► Edit   ai-harness.md ...             ← текущий шаг
```

### Поведение раскрытия

- **Во время выполнения** — раскрыт по умолчанию
- **После завершения** — сворачивается автоматически
- **Клик по заголовку** — переключает раскрытие в любой момент
- Состояние не сохраняется между сессиями

### Перевод системных событий

| `obj.subtype` (из stream-json) | Отображение | Иконка |
|---|---|---|
| `hook_started` | `Запуск` | `⚙` |
| `hook_response` | `Инициализация` | `⚙` |
| `init` | `Инициализация (${model})` | `⚙` |
| любой другой | сам subtype (фолбэк) | `⚙` |

### Счётчик в заголовке

Формат: `N шагов · X.Xs`  
Обновляется тикером (500 мс) во время выполнения.  
После завершения — финальные значения, не тикает.

`metricsEl` (отдельный div под заголовком) — убирается, счётчик переезжает внутрь `<h4>`.

## Изменения в коде

### `src/view.ts` (~30 строк)

**Новые поля:**
```typescript
private progressToggle!: HTMLElement;   // стрелка ▶/▼
private progressCount!: HTMLElement;    // "N шагов · Xs"
private stepsOpen = true;               // текущее состояние
```

**`onOpen()`:**
- `progressHeader` превращается в кликабельный контейнер
- `<h4>` содержит: стрелку + текст "Ход выполнения" + счётчик
- `stepsEl` — дочерний блок, скрывается/показывается по `stepsOpen`
- `metricsEl` — удаляется

**`setRunning()`:**
- `stepsOpen = true` + показываем `stepsEl`
- обновляем стрелку на `▼`

**`finish()`:**
- `stepsOpen = false` + скрываем `stepsEl`
- обновляем стрелку на `▶`

**`appendEvent()` для `ev.kind === "system"`:**
- маппинг subtypes → человекочитаемый текст
- иконка `⚙` вместо точки `·`

**`updateMetrics()`:**
- пишет в `progressCount` вместо `metricsEl`

### `src/stream.ts` — не трогаем

Перевод системных событий живёт во view, stream остаётся чистым парсером.

1. Блок раскрыт при старте
2. Системные события показывают переведённые строки
3. Счётчик обновляется в реальном времени
